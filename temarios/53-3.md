

## ⛓️ Cadenas de Razonamiento: *Pipelines* de Modelos Especializados

Las Cadenas de Razonamiento estructuran una tarea compleja en un **grafo de ejecución** donde el *output* de un modelo especializado se convierte en el *input* (o la evidencia) para el siguiente modelo de la secuencia.

Este enfoque es fundamental para tareas de alto riesgo (ej. diagnóstico o control de sistemas) donde la fiabilidad, la trazabilidad y la corrección deductiva son esenciales.

### 1. El Marco General de las Cadenas

Una cadena de razonamiento típica opera en tres o más fases, cada una manejada por el modelo más apto:

1.  **Percepción/Extracción de Datos:** Tareas probabilísticas (Neuro).
2.  **Inferencia/Predicción:** Tareas que requieren cómputo o modelos causales (Especializado/Simbólico).
3.  **Verificación/Explicación:** Tareas de razonamiento y comunicación (LLM/Simbólico).



### 2. Ejemplos de Cadenas de Razonamiento Especializadas

#### A. Cadena para el Diagnóstico (Integración Neuro-Simbólica)

| Módulo | Tipo de IA | Función en la Cadena |
| :--- | :--- | :--- |
| **1. Detector de Patrones** | *Deep Learning* (CNN/Visión) | Analiza datos crudos (ej. imágenes de resonancia magnética) y extrae características probabilísticas (ej. "Probabilidad de Masa Tumoral: 85%"). |
| **2. Formalizador** | LLM/IA Simbólica | Traduce el *output* probabilístico a hechos simbólicos (`has_tumor(patient)`) y extrae síntomas adicionales del historial médico del paciente (texto libre). |
| **3. Motor de Inferencia** | *Knowledge Graph* / Red Bayesiana | Utiliza la evidencia formalizada para calcular la probabilidad abductiva de las posibles enfermedades y propone el diagnóstico más probable. |
| **4. Explicador** | LLM | Recibe el diagnóstico y la prueba lógica del *Knowledge Graph*, generando una explicación natural, referenciada y trazable para el médico. |

#### B. Cadena para el Control Robótico (Planificación a Largo Plazo)

| Módulo | Tipo de IA | Función en la Cadena |
| :--- | :--- | :--- |
| **1. Comprensión de la Tarea** | LLM | Descompone el objetivo de alto nivel (ej. "Preparar un sándwich") en una secuencia de sub-metas abstractas (HTP). |
| **2. Planificador de Secuencia** | Modelo Causal / SAT *Solver* | Verifica que la secuencia de sub-metas sea lógicamente válida (ej. verificar que se debe "cortar" el pan antes de "untar" la mantequilla), garantizando la corrección deductiva. |
| **3. Ejecutor Local** | *Reinforcement Learning* (RL) | Traduce cada sub-meta a una secuencia de comandos de bajo nivel (ej. movimiento del brazo). Si falla, el *output* del sensor activa la replanificación. |
| **4. Supervisor (Self-Correction)** | LLM / Percepción | Monitorea si la ejecución cumplió la sub-meta. Si no (ej. "el pan se cayó"), la información de fallo se envía de vuelta al Planificador de Secuencia para generar una nueva estrategia. |

### 3. Beneficios de las Cadenas de Razonamiento

#### A. Trazabilidad y Explicabilidad
Al separar la tarea, se puede identificar exactamente qué módulo causó un fallo. El *output* de cada módulo es un artefacto intermedio (un "pensamiento") que justifica el siguiente paso.

* **Ejemplo:** Si el diagnóstico es incorrecto, sabemos si el fallo fue la **Percepción** (el detector de la imagen falló), la **Formalización** (el LLM tradujo mal un síntoma), o la **Inferencia** (la regla en el *Knowledge Graph* era errónea).

#### B. Robustez al Dominio
Cada modelo puede ser optimizado para su tarea específica y el tipo de dato que maneja (imagen, texto, código, reglas lógicas), aumentando la fiabilidad global. Si la tarea de percepción cambia ligeramente, solo es necesario actualizar el módulo de detección, no todo el sistema.

#### C. Integración de *Tool-Use*
La cadena de razonamiento es la implementación estructural del **Diseño de *Tool-Use* Sofisticado**. Cada modelo especializado en la cadena es esencialmente una herramienta experta que el LLM (el Planificador Global) invoca en el momento adecuado.

### 4. Conclusión

Las **Cadenas de Razonamiento** son la arquitectura preferida para la IA autónoma, ya que permiten la resolución de tareas complejas mediante la orquestación de **modelos especializados**. Al dividir el problema en pasos secuenciales (Percepción, Inferencia, Verificación) y delegar cada paso al motor de IA más robusto para esa función, se logra un sistema que no solo es más potente y adaptable, sino también inherentemente **trazable y explicable**.
